Opciones del comando nc
El comando nc acepta, entre otras, las siguientes opciones básicas:

-l: Sirve para que Netcat abra un puerto y se mantenga ala escucha. Se aceptará una única conexión de un único cliente antes de cerrarse.
-k: Se usa junto con la opción -l con el objetivo de que el puerto ser mantenga abierto tras recibir una conexión, a la espera de más conexiones.
-u: Permite abrir puertos con el protocolo UDP en vez de abrirlos mediante el protocolo TCP.
-p: Esta opción permite especificar el puerto al que conectarse.
-v: Se usa para mostrar información acerca de la conexión.
-t:  Respuestas compatibles con sesiones de Telnet.

La dirección IP 255.255.255.255 y la dirección IP 0.0.0.0 son diferentes en su significado y uso en una red:

255.255.255.255:

Definición: Esta dirección IP se refiere a la dirección de broadcast en una red.
Uso: Cuando se envía un paquete a la dirección 255.255.255.255, se envía a todos los dispositivos conectados a la red. Es decir, es una forma de enviar un mensaje a todos los dispositivos de la red local.
Ejemplo: Si un dispositivo quiere descubrir otros dispositivos en la misma red, puede enviar un paquete a 255.255.255.255 para que todos los dispositivos conectados a la red puedan recibirlo.
0.0.0.0:

Definición: Esta dirección IP se utiliza como una dirección de origen especial en ciertos contextos de configuración de red.
Uso: En muchos casos, la dirección 0.0.0.0 se utiliza para indicar "cualquier dirección" o "cualquier interfaz" en la configuración de red. Por ejemplo, en algunos casos de enrutamiento o configuración de sockets, 0.0.0.0 puede indicar que el dispositivo está dispuesto a aceptar conexiones en cualquier interfaz de red disponible.
Ejemplo: En algunos servidores, al configurarlos para escuchar en la dirección 0.0.0.0, están dispuestos a aceptar conexiones entrantes en todas las interfaces de red disponibles en el dispositivo.
En resumen, 255.255.255.255 se utiliza como dirección de broadcast para enviar mensajes a todos los dispositivos en una red, mientras que 0.0.0.0 se utiliza como una dirección especial en la configuración de red para indicar "cualquier dirección" o "cualquier interfaz".
